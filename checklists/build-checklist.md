# Build Checklist

Use this checklist when assembling or modifying a robot body for Asimov District.

## Planning

- [ ] Build purpose defined
- [ ] Robot type selected
- [ ] Expected tasks listed
- [ ] Risk class estimated
- [ ] Budget range defined
- [ ] Parts sourcing plan created
- [ ] Required tools listed

## Parts

- [ ] Bill of materials created
- [ ] Critical parts identified
- [ ] Spare parts identified
- [ ] Vendor links recorded
- [ ] Lead times recorded
- [ ] Compatibility checked
- [ ] Power requirements checked

## Assembly

- [ ] Frame assembled
- [ ] Actuators installed
- [ ] Sensors installed
- [ ] Cameras installed if needed
- [ ] Battery system installed
- [ ] Wiring checked
- [ ] Fasteners checked
- [ ] Moving parts checked for pinch hazards

## Software and Firmware

- [ ] Firmware installed
- [ ] Firmware version recorded
- [ ] Control software installed
- [ ] Network settings configured
- [ ] Update procedure documented
- [ ] Rollback procedure documented

## Calibration

- [ ] Motors calibrated
- [ ] Sensors calibrated
- [ ] Cameras calibrated
- [ ] Balance or stability checked
- [ ] Movement range checked
- [ ] Speed limits configured

## Safety Controls

- [ ] Local emergency stop installed
- [ ] Remote shutdown configured
- [ ] Operator override configured
- [ ] Safe idle behavior configured
- [ ] Power isolation procedure documented
- [ ] Network disconnect procedure documented

## Documentation

- [ ] Build photos captured
- [ ] Wiring notes captured
- [ ] Parts manifest updated
- [ ] Firmware version logged
- [ ] Known issues documented
- [ ] Maintenance schedule created

## Handoff

- [ ] Body registry record created
- [ ] Safety test scheduled
- [ ] Operator training assigned
- [ ] Deployment checklist not yet approved until safety arena pass

## Build Rule

A robot build is not complete until the body can be safely stopped, inspected, repaired, and audited.
